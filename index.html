"Stacking with z-index"

5/6/25

In this short blog post, we will be briefly examining the concept of the z-index property in HTML and a simple example of it’s function in application. This post is by no means comprehensive and is intended only as a general guide. Additional research is strongly encouraged.

On your computer screen, objects are shown horizontally along the X axis, vertically along the Y axis and “three-dimensionally” along the Z axis. Also known as “Stacking Context,” the z-index property is a three-dimensional design tool for HTML which permits page elements to be stacked one atop another along the web browser’s Z axis. This allows for elements to be “closer” or “further” from the user, above or under other elements, just as if they were sheets of paper laid over each other on a surface.

The z-index property can be set using a numerical value, either positive or negative. Elements set with a higher value (or greater “stack order”) will be displayed above those with lower values (or lower “stack order”). However, if there is a case of overlapping elements where no z-index value is given, then the elements will be rendered automatically in an order determined by the position of each element in the markup document. The element in the lowest part of the document would in this case be displayed on top, being the most recent, above any elements preceding it in the code.

The z-index property will only work on positioned elements, such as when the CSS position property is set to “absolute,” “relative,” “fixed,” or “sticky” and elements which are the direct children of elements having their display property set to “flex.” Leaving out other properties for sake of demonstration, examples of positioned elements in CSS would be: 

                        
.item1 {
    position: absolute;
    z-index: 1;
}

.item2 {
    position: relative;
    z-index: 2;
}

.item3 {
    position: fixed;
    z-index: 3;
}

.item4 {
    position: sticky;
    z-index: 4;
}

.item5 {
    display:flex;
    z-index: 5;
}

Suppose now that we wanted to draw three shapes in an overlapping formation to demonstrate the functionality of the z-index property in a simple example. Our code might look something like the following: 

                        
<style>
    .zdemo {
        position: relative;
    }
    
    .item1 {
        position: relative;
        z-index: 1;
        border: 5px solid #FFF;
        height: 200px;
        margin: 40px;
    }
    
    .item2 {
        position: absolute;
        z-index: 2;
        background: #F00;
        color:#000;
        width: 35%;
        height: 140px;
        right: 30%;
        top: 100px;
    }
    
    .item3 {
        position: absolute;
        z-index: 3;
        background: #00F;
        width: 50%;
        height: 40px;
        left: 300px;
        top: 130px;
    }
    </style>
    
    <h6>"Stacking with z-index"</h6>
    
    <div class="zdemo">
        <div class="item1">Item 1 with "z-index: 1"</div>
        <div class="item2">Item 2 with "z-index: 2"</div>
        <div class="item3">Item 3 with "z-index: 3"</div>
</div>

Here we have three items placed inside a div, fully styled and positioned (note the X and Y axis properties such as “right,” “left,” “top,” and so on). For the purposes of this example, the artistic properties of each item (shape, color, etc.) are unimportant. Take note, however, that each of the three items has the “z-index” property followed by a value (in this case 1, 2 and 3). As we have read, these values will determine as noted above the placement of each item in the Z axis hierarchy. This example code produces the following result:

"Stacking with z-index"
Item 1 with "z-index: 1"
Item 2 with "z-index: 2"
Item 3 with "z-index: 3"

With the example code now rendered, we can now see how the z-index value of each item produces the resulting stack order. Each item took the desired place in the layering process. Of course, z-index is useful for far more than mere placement of shapes. Z-index can be used for many creative or practical purposes. The navigation bar of this page, for example, is kept above all other elements using this method.

Now that you have had a brief introduction, try to apply what you might’ve learned here about the z-index property in one of your own projects. Have fun!
